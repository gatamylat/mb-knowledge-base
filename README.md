# МБ База Знаний (ТехЧекПро)

Одностраничная PWA — свод норм мебельного производства «Массивбург».
Один HTML-файл + корпус + папка сторис. Никаких внешних зависимостей кроме шрифтов.

## Состав репозитория

```
index.html            — само приложение (HTML + CSS + JS + IndexedDB + движок сторис)
mb_corpus.json        — корпус из 206 записей (стартовый)
stories/
  index.json          — манифест: список сторис (правишь его, добавляя сторис)
  zerkalo.json         — пример сторис «Работа с зеркалом»
```

## Развёртывание (GitHub Pages)

1. Создать репозиторий, положить в корень `index.html`, `mb_corpus.json` и папку `stories/`.
2. Settings → Pages → ветка `main` → Save.
3. Через 1–2 минуты приложение доступно по адресу GitHub Pages.

Локально — открыть `index.html` двойным кликом (сторис и корпус подгрузятся из соседних файлов; на file:// часть fetch может не работать — для полной проверки нужен GitHub Pages или локальный сервер).

> Если в браузере висит старый корпус — меню (⋯) → «Перезагрузить стартовый корпус».

## Три способа работы с базой

Переключатель в шапке: **Лента | Книга**. Плюс **Сторис** — кружками над Лентой.

### Лента — повседневный поиск
Карточки, мгновенный поиск с русской морфологией, фильтры по категориям и типам,
значок «фото» на карточке. Клик → полная запись.

### Книга — чтение подряд
Обложка → оглавление → главы по логике регламента. Фото под заголовком записи.

### Сторис — тематические рассказы (кружки над Лентой)
Кружки в стиле «историй»: непросмотренные с цветным кольцом, просмотренные серые
и уезжают в конец. Первый пунктирный кружок «Смотреть все» проигрывает непросмотренные
подряд. Тап по кружку → полноэкранный проигрыватель (свайп / тап-зоны / стрелки).
Просмотренные не исчезают — гаснут, но доступны для пересмотра.

## Как добавить новую сторис

1. Собрать JSON-файл сторис (вместе с Claude), положить в `stories/`, например `stories/metall.json`.
2. Дописать строку в `stories/index.json`:
   ```json
   { "id": "metall", "file": "stories/metall.json", "title": "Узлы по металлу", "icon": "М" }
   ```
3. Готово — кружок появится над Лентой. **index.html трогать не нужно.**

Браузер не умеет читать содержимое папки на сервере, поэтому список ведётся в манифесте
`stories/index.json`. Это и есть «папка, которая сама себя описывает».

### Формат файла сторис
```json
{
  "id": "zerkalo",
  "cover": { "brand": "ТехЧекПро", "tag": "Стандарты",
             "title": "Работа с зеркалом", "sub": "4 шага" },
  "slides": [
    { "ref": "id_записи_корпуса" },                    // тянет title/текст/фото из корпуса
    { "tag": "...", "title": "...", "body": "...",      // или свой текст слайда
      "insight": "почему так" }
  ],
  "final": { "title": "...", "sub": "..." }
}
```
Слайд может **ссылаться на запись корпуса** по `"ref": "id"` (тогда заголовок, текст и фото
подтягиваются автоматически) **или нести свой текст** (`title`/`body`/`insight`).

## Фотографии узлов

К записи можно прикрепить одно фото (схема/чертёж/снимок). Только «как надо».
- Прикрепляется в форме записи, сжимается до ~1400px.
- Хранится в **IndexedDB** (`mb_kb_media`), а не в localStorage — до сотен МБ.
- В JSON у записи только флаг `hasPhoto: true`.
- Клик по фото — полноэкранный лайтбокс.

> ⚠️ Экспорт JSON сохраняет только текст (и флаг `hasPhoto`), но **не сами фото** —
> они в IndexedDB конкретного браузера. При переносе на другое устройство фото
> прикрепляются заново. Синхронизация фото — задача следующего этапа.

## Контроль конфликтов
При создании похожей записи — модалка «Возможный конфликт». Меню → «Поиск конфликтов
в базе» находит группы пересекающихся норм. Устаревшие помечаются и прячутся.

## Экспорт / импорт
Меню (⋯): экспорт JSON, экспорт в Word (.docx), импорт JSON, перезагрузка корпуса, очистка.

## Тёмная тема, установка, хоткеи
Тема — кнопка-полумесяц. Установка как PWA — «добавить на главный экран».
Ctrl/⌘+K — поиск, Esc — закрыть модалку/лайтбокс/проигрыватель.

## Технологии
- Один HTML-файл, без сборки и зависимостей (кроме Google Fonts).
- Текст — localStorage; фото — IndexedDB.
- Движок сторис: проигрыватель в ядре, сторис — данные в `stories/*.json`.
- Дизайн — Surf Design System (тёплый книжный, без чисто белого/чёрного).
